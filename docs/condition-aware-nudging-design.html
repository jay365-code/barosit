1. 배경 & 문제의식

BaroSit은 이미 카메라에서 Pose 33점(전신 뼈대) + 정밀 얼굴 메시(468점) + 머리 각도(pitch/yaw/roll)를 실시간으로 뽑고 있다. 지금은 이 신호를 자세 판정에만 쓴다. 하지만 같은 신호를 시간축으로 누적하면 사용자의 컨디션(집중/딴짓/졸음/피로)을 추정할 여지가 크다.

모니터링 중 발생하는 "업무 이탈"의 현대적 패턴은 대부분 핸드폰(유튜브·SNS) — 즉 시선/고개가 아래로 떨어지는 행동이다. 정밀한 좌우 시선 추적보다, "떨어짐 + 지속"을 보는 것이 기술적으로 더 견고하고 실사용 패턴과도 맞는다.

제약 상수: 온디바이스 · 카메라 단일 · 추가 모델 도입 최소화(기존 468점·pitch 재활용) · 임상 주장 금지(휴리스틱 추정치로 포지셔닝).

2. 핵심 컨셉

흩어진 아이디어(딴짓/시선이탈/졸음/피로도)는 사실 하나의 파이프라인으로 수렴한다:

얼굴·자세 신호 (실시간) │ ▼ [시간축 누적 · 개인 기준선 대비] 컨디션 신호 추정 ── 딴짓 / 시선이탈 / 졸음 / 피로 │ ▼ [가중 합성] 컨디션 인덱스 (0~1, 상대값) │ ▼ [적응 변수로 주입] 기존 JITAI 넛지 엔진 ── 빈도·강도·톤·타이밍·휴식권유 조절

즉 새 기능 4개가 아니라, "컨디션 인지 레이어" 1개 + 기존 넛지 엔진의 적응화다.

3. 감지 신호 4종

3-1. 딴짓 (Distraction) — 고개 아래로 떨어짐 지속

정의: 기준자세 대비 pitch가 아래로 X° 이상 + Y초 이상 지속 (얼굴 여전히 present).

왜 견고한가: pitch-down은 측면 45° 카메라에서도 살아남는다(좌우 시선/홍채는 무너짐). 실제 딴짓 = 무릎/책상 폰 내려다봄 = pitch-down 지속.

오탐 처리: 짧게 내려다봄(키보드·서류)은 지속시간 임계값으로 필터. "서서히 숙임(자세)" vs "오래 아래 고정(딴짓)"은 지속성/패턴으로 분기.

pitch 계산 보유 지속-딴짓 판정 신규

3-2. 시선 이탈 (Gaze-off) — 거친 수준만

정의: yaw가 "작업 기준선"에서 크게 벗어나 지속 (딴 데로 고개 돌림).

한계: "어느 화면을 보는지" 정밀 판별은 45° off-axis에서 불가. 거친 present/away · 큰 고개돌림까지만 목표.

yaw · MultiAngle 기준선 보유 이탈 지속 판정 확장

3-3. 졸음 (Drowsiness)

하위 신호방법측면 45°상태
눈 감김 / PERCLOS눈 랜드마크 EAR, 또는 eyeBlink blendshape약함(먼 눈 압축)확장
깜빡임 느려짐/길어짐EAR 시계열약함확장
하품입 벌림(MAR) / jawOpen견고확장
고개 꾸벅(microsleep nod)pitch 급강하→반등견고확장

현재 얼굴 모델은 outputFacialTransformationMatrixes:true만 켜져 있음 (src/pose/detector.ts:143). outputFaceBlendshapes:true 한 줄이면 eyeBlink/jawOpen 값이 바로 나와 EAR/MAR 수동계산을 대체·보강한다.

측면 세팅 전략: 눈 신호가 약하므로 하품 + 고개꾸벅 조합을 주력, 눈 감김은 보조.

3-4. 피로도 (Fatigue) — 합성 지표

단일 신호로는 안 잡힘. 여러 대리 신호를 개인 기준선 대비 롤링 누적·가중합.

신호피로와의 관계상태
작업 지속 시간가장 강한 프록시 (오래 = 피로↑)세션 추적 보유
자세 붕괴 속도지칠수록 구부정·기대기 위반 빈도↑violation 추적 보유
움직임 변동성 변화피로 시 미동↓(축 처짐) 또는 잔움직임↑variabilityTracker 보유
깜빡임/PERCLOS피로 초기 신호확장(측면 약함)
하품 빈도 증가세션 후반 상승확장
고개 꾸벅 빈도후기 피로/졸음확장

variabilityTracker: 롤링 10분 윈도우로 어깨Y·코Y/Z·face pitch 표준편차 = movement index — src/pose/variabilityTracker.ts

4. 피로 인덱스 설계

목표: 절대 측정이 아닌 "자기 자신 대비 상대 피로 추정치" (0~1).

fatigueIndex(t) = w1 · timeOnTaskNorm // 마지막 휴식 이후 경과 (자연 상승) + w2 · postureDecayRate // 최근 위반 빈도 상승 기울기 + w3 · variabilityDrift // movement index의 기준선 대비 이탈 + w4 · ocularFatigue // PERCLOS/blink (측면이면 가중치↓) + w5 · yawnRate + w6 · nodEvents → EMA 평활 → 0~1 clamp
  • 개인 기준선: 세션 초반 N분을 "정상 컨디션"으로 잡고 이후를 상대 비교 (개인차 흡수).
  • 가중치 적응: 측면 세팅 감지 시 w4(눈)↓, w1·w2·w3(시간·자세·변동성)↑.
  • 표시: 숫자 노출은 신중히 — "피로 %"보다 3단계(양호/누적/휴식필요) 정도의 정성 표현 권장.

5. 적응형 개입 (JITAI 연동)

BaroSit엔 이미 JITAI·백오프·보상 넛지 슬라이스가 있다(docs/posture-nudge-design.md). 컨디션 인덱스를 그 엔진의 입력 변수로 주입하면 "적응형"이 자연히 완성된다.

컨디션 상태개입 적응
낮은 피로 · 집중넛지 최소화 · 방해 안 함 (몰입 보호)
피로 상승 중스트레칭·자세 넛지 타이밍 강화 · 톤 부드럽게
딴짓 지속 감지집중 모드에서 가벼운 리마인드 (자세 잔소리와 톤 분리)
졸음 신호(꾸벅+하품)"잠깐 일어나 걷기/세수" 권유 · 자세 넛지는 유예
높은 피로자세 잔소리 대신 진짜 휴식 권유 · 넛지 빈도↓ (역효과 방지)
적응형 포모도로: 피로가 오르면 작업 블록을 자동 단축 — 고정 25분이 아니라 컨디션 기반 가변 사이클.

6. 측면 45° 세팅 대응

노트북이 좌/우 45°에 있고 서브모니터(정면)를 응시하는 흔한 세팅. 카메라 입장에선 얼굴이 ~45° 돌아간 상태 = 얼굴 랜드마크 성능 저하 시작점.

신호45°에서대응
pitch (딴짓·꾸벅·거북목)견고주력 신호로 채택
하품(입 벌림)견고졸음 주력
yaw (시선이탈)중간거친 판정만
눈 EAR/PERCLOS약함가중치↓, 보조로만
얼굴 유실(>~55°)추적 끊김pose 33점으로 몸 잡히면 "자리비움" 오판 유예 필요

가장 큰 실질 리스크: 고개를 서브모니터로 더 돌리면(55~60°) 얼굴 유실 → 앱이 "자리비움"으로 오판. 측면 사용자 UX의 핵심 이슈. → 얼굴 없어도 pose 몸 신호로 present 유지하는 유예 로직 확인/보강 필요.

BaroSit은 이미 MultiAngleBaseline(front/left/right) + yaw 히스테리시스가 있어(src/pose/calibration.ts) 측면을 부분 대응하나, 전환 임계가 yaw ±12°라 45° 고정이면 항상 left/right 기준으로 평가됨 → 그 쪽 기준자세 캘리브레이션 필수.

권장: 정밀 컨디션 감지가 필요하면 서브모니터 위 외장 웹캠(정면)이 최선. 노트북 45° 유지 시엔 "거친 딴짓/졸음(꾸벅·하품) + 시간·자세 기반 피로"까지가 현실적 목표.

8. 한계 · 리스크 · 원칙

  1. 검증 불가 문제: 카메라 기반 피로/졸음은 임상 측정이 아닌 휴리스틱 추정. "피로 감지"가 아니라 "컨디션 케어/추정 신호"로 포지셔닝. 메모리의 임상주장 인용규칙(PRI/27kg 금지) 준수.
  2. 개인차: 피로·깜빡임·자세 기준선은 사람마다 극단적. 자기 기준선 대비 상대 판정 필수, 절대 임계 금지.
  3. 신뢰/거부감: 카메라로 인지·정서 상태 추론은 민감. 온디바이스가 큰 무기 — 투명하게 opt-in, 데이터 미전송 명시.
  4. 개입 역효과: 적응형 넛지가 틀리면 짜증·조종당하는 느낌. 보수적 임계 + 사용자 컨트롤 + 백오프 필수.
  5. 포지셔닝 흐림: BaroSit은 자세 코치. 졸음/피로는 인접하나 결이 다름 → "집중/컨디션 케어" 부가 레이어로 묶되 주 정체성을 흐리지 않기.

9. 단계적 구현 계획 (제안)

단계내용비용
P0 · 딴짓pitch-down 지속 판정 + 집중모드 리마인드. 측면에서도 견고, 가치 즉시 체감.신규 소

P0만으로도 독립적 가치가 있어 "작게 시작 → 반응 보고 확장"이 가능하다.

10. 열린 질문

  • 딴짓/자세 위반을 같은 신호로 볼지, 분리할지 — 톤·개입이 달라짐.
  • 피로 인덱스를 사용자에게 노출할지 (동기부여 ↔ 불안·자기감시 유발).
  • 졸음/피로가 BaroSit 정체성에 부가 가치인지, 산만인지 — 유료 차별화 요소로 쓸지.
  • 기업 도입(B2B) 시 "직원 감시" 프레임 리스크 — 개인 온디바이스 원칙을 어떻게 계약·UX로 보장할지.
  • 정면 웹캠 사용자와 측면 사용자에게 다른 기능 셋을 줄지(감지 신뢰도 차이).
