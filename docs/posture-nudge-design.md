## 1. 주기/간격 — 검증된 근거 (RCT/메타분석)

| 결과 | 권장값 | 출처(검증 통과) |
|---|---|---|
| 8h RCT(n=11) 30/60분 × 1/5분 걷기 휴식 비교 | — | [Breaking Up Prolonged Sitting](https://exerciseismedicine.gr/wp-content/uploads/2023/05/Breaking_Up_Prolonged_Sitting.pdf) 3-0 |
| 혈당: "30분마다 5분"만 유의 감소(iAUC −11.8, P=.017) | **30분 / 5분 = 혈당 최적** | 동일 3-0 |
| 혈압: 4개 용량 모두 유의 감소(60분/1분도 −5.2mmHg) | 짧고 드물어도 혈압엔 충분 | 동일 3-0 |
| 피로/기분: 5분 > 1분 (1분은 약함/무의미) | **길이는 5분 > 1분** | 동일 2-0 |
| 네트워크 메타분석: "30분마다" 혈당/인슐린 개선 최적(SUCRA 81.8/77.5%), 20분도 유의 | **20–30분 스위트스팟** | [MDPI 14:3201](https://www.mdpi.com/2076-3417/14/8/3201) 2-0 |

**종합:** "**20–30분마다, 약 5분의 가벼운 움직임(걷기)**"이 가장 근거 강한 권장값. 정확한 분 단위보다 *자주(20–30분) + 충분히 길게(>1분, 이상적 5분)*가 핵심. (미검증 반례: "6분/60분 > 3분/30분" 등 정확 최적점엔 노이즈 존재 → 단정 회피.)

### 현재 BaroSit 기본값 진단
- micro **30분** → ✅ 근거 일치. 유지.
- standup 50분 / deep 120분 → 합리적 사다리. 유지.
- variability **10분** → 대사 휴식과 별개 축(정체 감지). 30분 휴식 전 *선행 가벼운 넛지*로 layering 자연스러움. 유지.
- ⚠️ 갭: micro 권장 액션이 "수초"급 → 근거는 5분 가벼운 활동이 더 낫다. **"짧아도 OK(혈압)" 유지 + "이왕이면 2–5분 걷기" 권장으로 상향** 여지.

## 2. 강제 vs 넛지 / 순응도 — ✅ 검증된 근거 (2026-06-22 3차 리서치)

**(1) 강제는 단기 순응도↑, 장기 이탈·해악↑ → 기본 강제 금지, 옵트인만.**
- 하드스톱: CPOE RCT 57.2% vs 13.5%, 중복검사 하드스톱 92.3% vs 소프트 43.6% (단기 순응도 급증)
- 그러나 32개 연구 중 11개서 해악, 한 RCT는 환자 해악 4건으로 **조기 중단** — 저자들 하드스톱 비권장
- 수동 푸시도 단기 효과 큼(1시간 내 앱오픈 3.52배)이나 장기 이탈은 못 막음
- → 강제 중단은 **옵트인 전용**, 기본은 부드러운 유도. ([Strom 2010](https://pubmed.ncbi.nlm.nih.gov/20876410/), [Tsai 32-study review](https://pmc.ncbi.nlm.nih.gov/articles/PMC6915824/))
- (참고: "하드스톱이 순응도 100%까지" 주장은 3차에서 **반박됨** 0-3)

**(2) 순응도는 시간이 갈수록 하락 → 무시율 가속 = 이탈 조기경보.**
- EMA 메타(477연구) 평균 79%; 14주에 88.9%→70%; 예측가능 알림 ~2%/주·랜덤 ~1%/주 감소
- 저참여자가 가장 빨리 이탈(드롭아웃 10%/주 vs 완주자 2%/주) → **무시 가속을 churn 신호로** ([EMA meta](https://pmc.ncbi.nlm.nih.gov/articles/PMC9999286/), [JMIR 2023](https://www.jmir.org/2023/1/e43826))

**(3) 맥락 인식(JITAI)이 고정 스케줄보다 순응도 유지에 우월.**
- 맥락 인식 알림 응답률↑(pooled OR 1.71); 센서 추론 타이밍 > 사용자 지정 정적 시간(후자는 효과 無)
- **알림 빈도(개수)는 예측력 없음**(b=0.008, p=.714) — 절대 빈도보다 **맥락 적합·개인화**가 중요
- → BaroSit은 카메라로 "방해해도 되는 순간"을 감지 가능 = JITAI 차별점 ([context meta](https://pmc.ncbi.nlm.nih.gov/articles/PMC9494762/))

**(4) 알림 끄기는 행동을 못 바꾼다; 보상 > 처벌(자율성 지지).**
- 1주 알림 비활성 RCT(N=205): 객관 행동 불변, 인지 습관만↓·FOMO↑ → 억제는 답이 아님
- 자기결정이론: 처벌·죄책감은 자율성 훼손→이탈 가속 → **보상 중심 설계** (효과크기 직접치는 미검증, 원칙 기반) ([RCT](https://www.tandfonline.com/doi/full/10.1080/15213269.2024.2334025))

## 3. 확정 설계 — 점증 에스컬레이션 + 긍정강화

하드 강제(화면 잠금)는 **기본값 아님**. 준수 감지 기반 점증 + 긍정 강화.

```
변동/휴식 주기 도달
  ├─ 1단계 부드러운 넛지 (현행 청록 카드 유지)
  │     └ [준수창 30~60초] 움직임/스트레칭/일어서기?
  │           ├ YES → ✅ 긍정강화: 칭찬 + 점수/스트릭 보너스 + 종료
  │           └ NO  → 2단계
  ├─ 2단계 약간 강한 리마인더 (조금 크게/오래, 1~2분 후)
  │           ├ YES → 축소 보상 + 종료
  │           └ NO  → 3단계
  └─ 3단계 단호하지만 비잠금 ("스누즈 5분 / 지금 하기")
              └ 계속 무시 → ❌죄책감 X, 조용한 백오프 + 시간대 패턴 학습(적응형 빈도↓)
```

원칙 (전부 §2 근거로 뒷받침):
- **하드 잠금 = 옵션(집중모드)만.** 기본 강제 X → 반발/장기 이탈·해악 회피 [§2-(1)].
- **에스컬레이션은 "더 시끄럽게"가 아니라 "더 좋은 순간에".** 빈도(개수)는 예측력 없음 → 강도 경쟁보다 **맥락 적합 타이밍(JITAI)** 우선. BaroSit은 카메라로 방해 가능 순간(위반 사이·전환 시점) 감지 가능 [§2-(3)].
- **따랐을 때 = 즉각 긍정강화** (기존 "잔소리 대신 응원" 톤·스트레칭 보너스와 연결) [§2-(4)].
- **안 따랐을 때 = 처벌/죄책감 X, 조용한 백오프 + 적응형 빈도.** 단순히 알림을 끄는 게 아니라 *타이밍·품질*을 조정 [§2-(2),(4)].
- **무시율 가속 = 이탈 조기경보**로 활용 — 백오프뿐 아니라 "톤/방식 전환" 트리거 [§2-(2)].

## 4. 구현 — 코드 빈칸 3가지 + 집중모드

1. **준수 로깅** — ✅ **구현 완료** ([complianceTracker.ts](../src/pose/complianceTracker.ts))
   - 알림 발사 후 응답 윈도우(60초) 내 움직임(스트레칭/기립/휴식/자리비움, 변동성은 자세변동도) 감지 → 준수/미준수 확정
   - 순응률·연속무시(ignoreStreak) 노출 + 일일 집계 영속화(`nudge_compliance_today`)
   - 두 루프(MonitorView·useMonitoringEngine) 배선, 단위테스트 7개
2. **보상 루프** — ✅ **구현 완료**
   - 준수 시 점수 보너스(+2, `posture-bonus`) + 긍정 토스트([AlertOverlay](../src/components/AlertOverlay.tsx) 초록 "좋아요! 잘 움직였어요", ko/en/ja)
   - 미준수는 죄책감/처벌 X — 조용히 집계만
3. **적응형 백오프** — ✅ **구현 완료** (`computeBackoffMultiplier`)
   - `ignoreStreak` 소비 → 변동성 알림 쿨다운 ×(1.0~2.0) 확대. streak 0→1.0, 4+→2.0
   - 두 루프에 배선(매 프레임 status()로 backoff 산출 후 cooldownMinutes 조정), 테스트 3개
   - **점증 강도는 의도적으로 약하게** — §2-(3) "빈도/강도보다 타이밍" → 강도 경쟁 대신 빈도 백오프 채택
   - (휴식 알림은 1회/스테이지 발사라 빈도 백오프 대상 아님 — 변동성에만 적용)

4. **JITAI 발사 타이밍** — ✅ **구현 완료** ([jitaiGate.ts](../src/pose/jitaiGate.ts))
   - 휴식 알림을 즉시 안 쏘고 **방해 가능 순간(고개 돌림 yaw>0.26rad · 움직임)** 까지 보류했다 발사. 좋은 순간 없으면 maxHold(90초) 후 발사. 사용자가 그새 휴식 취하면 보류 폐기
   - 두 루프 배선, 변동성 알림엔 미적용(트리거가 "정지"라 부적합), 단위테스트 7개
   - [§2-(3)] "빈도/강도보다 타이밍"(OR 1.71) 직접 구현 — BaroSit 카메라 = 핵심 차별점

5. **집중모드(옵트인 에스컬레이션)** — ✅ **구현 완료**
   - AlertModes.focusMode(기본 OFF) + SettingsDrawer 토글(ko/en/ja). 무시된 *휴식* 알림에 한해 더 또렷한 중앙 프롬프트로 한 번 더 권유([AlertOverlay](../src/components/AlertOverlay.tsx) escalation 카드, 8초, 주황)
   - **화면 잠금 같은 하드 강제 아님** — "단호하지만 닫을 수 있는" 비잠금. 죄책감 없는 카피. [§2-(1)] 근거상 기본 OFF·옵트인 전용
   - 두 루프에 배선(준수 resolved=미준수 & break_* & focusMode → dispatchEscalationAlert)
