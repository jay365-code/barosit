📌 이 문서 사용법 (에이전트 인계 가이드)

  • 각 항목의 상태는 <div class="item" data-status="..."> 의 data-status 속성으로 관리한다. 허용값: done(완료) · partial(부분) · todo(미구현) · verify(확인 필요).
  • 배지·색·상단 진행률은 data-status 를 읽어 JS가 자동 렌더. 작업을 끝내면 속성값 한 곳만 바꾸면 된다 (예: data-status="todo" → "done").
  • 항목을 완료로 바꿀 때 .next(다음 작업) 블록은 지우거나 검증 메모로 갱신하고, changelog.md · project-status.md 도 동기화한다.
  • 근거는 파일:라인 으로 표기. 코드와 불일치하면 코드를 신뢰하고 이 문서를 고친다.
  • 상단 칩(완료/부분/미구현/확인필요)을 클릭하면 해당 상태만 필터링된다.
  • 우선순위가 매겨진 갭은 ID 배지로 표시 (OPS-1 등). 권장 착수 순서는 맨 아래 우선순위 참고.
