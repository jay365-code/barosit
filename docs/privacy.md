# BaroSit 개인정보 처리방침

**최종 갱신일**: 2026년 5월 19일
**시행일**: 2026년 5월 19일

> ⚠️ 본 문서는 **초안**이며, 최종 출시 전 변호사 검토를 거쳐 확정합니다.

BaroSit(이하 "본 서비스")은 사용자의 자세 건강을 도와드리기 위해 웹캠 영상을
실시간 분석하는 데스크톱 앱입니다. 본 처리방침은 본 서비스가 어떤 정보를
어떻게 처리하는지를 투명하게 공개하기 위해 작성되었습니다.

본 서비스는 **「개인정보 보호법」**(대한민국) 및 **EU 일반 데이터 보호 규정
(GDPR)** 의 원칙을 따릅니다.

## 2. 핵심 원칙 — 온디바이스 처리

**웹캠 영상은 사용자의 컴퓨터를 절대 떠나지 않습니다.**

- 자세 인식(MediaPipe 모델)은 100% 사용자 기기 안에서 실행됩니다.
- 본 서비스는 **카메라 영상 프레임, 캡처된 사진, 비디오 녹화를 일절 수집·전송·저장하지 않습니다.**
- 인터넷이 끊긴 상태에서도 자세 감지·점수·알림은 정상 동작합니다.

## 3. 수집·처리하는 정보의 항목

### 3-1. 본 서비스가 사용자 기기에 저장하는 정보 (localStorage)

본 서비스는 다음 정보를 **사용자 기기의 브라우저 저장소(localStorage)** 에만
저장합니다. 본 서비스 운영자의 서버로 전송되지 않으며, 운영자는 접근할 수
없습니다.

| 항목 | 내용 | 키 |
|---|---|---|
| 캘리브레이션 기준 자세 | 사용자가 처음 측정한 어깨/머리 좌표(랜드마크 33점) | `calibration_baseline` |
| 자세 이벤트 이력 | 위반 시각·종류·지속시간(영상/사진 아님, 메타데이터만) | `posture_events` |
| 자세 점수 | 0–100 점수 + 마지막 갱신 시각 | `posture_score` |
| 자세별 임계값·민감도 | 사용자가 설정한 값 | `thresholds` |
| 휴식·누적·변동성·적응형 설정 | Phase 1–4 모듈 사용자 설정 | `break_config`, `cumulative_load`, `variability`, `adaptive_sensitivity` |
| 위젯 위치·상태·UI 설정 | 미니바 표시·테마·실루엣 모드 등 | `widget_position`, `widget_state`, `minibar_visible`, `app_mode` 등 |
| 스트레칭 카운트 | 오늘/어제 스트레칭 보너스 발사 횟수 | `stretches_today`, `stretches_yesterday` |
| Anthropic API 키 (선택) | 사용자가 LLM 코칭 활성화 시 입력 | `anthropic_api_key` |
| 알림 모드 / 온보딩 완료 / 업데이트 snooze 상태 | UX 상태 | `alert_modes`, `onboarded_v1`, `updater_snoozed_version` |

전체 키 목록은 [src/dataBackup.ts](../src/dataBackup.ts) 의 `BACKUP_KEYS` 배열에 명시되어 있습니다(소스 코드 공개).

### 3-2. 본 서비스가 외부로 전송하는 정보 — **사용자 동의 또는 명시적 활성화 시에만**

#### A. LLM 코칭 (기본 비활성)

- **활성화 조건**: 사용자가 설정 → "AI 코칭 메시지" 토글을 켜고 자신의 **Anthropic API 키** 를 입력한 경우에만 동작합니다.
- **전송 정보**: 영상이 아닌 **자세 텍스트 메타데이터** 만 전송합니다.
  - 자세 종류 (예: `slouching`)
  - 지속 시간 (초)
  - 오늘 같은 자세 위반 횟수
  - 현재 시각(시)
- **전송 대상**: Anthropic 사 (Claude API · https://www.anthropic.com)
- **운영자 서버 경유 여부**: 경유하지 않습니다. 사용자 본인 API 키로 사용자 기기에서 직접 호출합니다.
- **수신측 처리**: Anthropic 의 [Privacy Policy](https://www.anthropic.com/legal/privacy) 를 따릅니다.

#### B. 자동 업데이트 체크

- 앱 시작 시 약 5초 뒤 1회(마지막 체크 후 24시간 경과 시) 새 버전 메니페스트를 가져옵니다.
- **요청 대상**: `https://github.com/jay365-code/barosit/releases/latest/download/latest.json` (GitHub 인프라)
- **전송 정보**: 표준 HTTPS 요청 헤더(IP·User-Agent 등)만 GitHub 측에 기록될 수 있으며, 본 서비스 운영자는 수집·열람하지 않습니다.
- **수신측 처리**: GitHub 의 [Privacy Statement](https://docs.github.com/en/site-policy/privacy-policies/github-general-privacy-statement) 를 따릅니다.

#### C. MediaPipe 모델 다운로드 (첫 실행 시 1회)

- 자세·얼굴·손 인식 모델 약 20MB 를 Google CDN(`storage.googleapis.com`)에서 다운로드합니다. 이후 캐시되어 오프라인 사용이 가능합니다.
- **전송 정보**: 표준 HTTPS 요청 헤더. 본 서비스 운영자는 수집하지 않습니다.

### 3-3. 본 서비스가 수집하지 않는 정보

본 서비스는 다음 정보를 **일절 수집하지 않습니다.**

- 이름·이메일·연락처·계정 정보 (회원가입 자체가 없음)
- 카메라 영상 프레임·사진·비디오
- 키 입력·화면 캡처
- 위치 정보·IP 주소(운영자 서버로의 전송 없음)
- 분석·텔레메트리·크래시 리포트 (향후 옵트인 방식으로 도입 시 별도 고지)
- 광고 식별자

## 4. 보유 및 이용 기간

| 정보 | 보유 기간 |

운영자는 사용자 기기 내 데이터에 접근할 수 없으므로 운영자 보유 기간은 해당
없습니다.

## 5. 정보주체의 권리

본 서비스는 운영자 서버에 사용자 정보를 보관하지 않으므로, 다음 권리는
**사용자 본인이 앱 내에서 직접 행사** 합니다.

| 권리 | 행사 방법 |
|---|---|
| 열람 | 설정 → "데이터 → 내보내기" — JSON 파일로 전체 다운로드 |
| 정정 | 설정 → "기준 다시 잡기" / 각 설정 슬라이더 |
| 삭제 | 설정 → "기록 초기화" — 즉시 전체 삭제 |
| 처리 정지 | 메인 화면 일시정지 버튼 또는 앱 종료 |
| 이동(이식) | 설정 → "데이터 → 내보내기" 의 JSON 형식 |

GDPR 적용 사용자(EU 거주자)는 위 권리 외에도 EU 일반 데이터 보호 규정 제15–22조에 따른 권리를 본 서비스 운영자에게 직접 요청할 수 있으나, **운영자가 보유하는 개인정보 자체가 없습니다**.

## 6. 데이터 보안

- **온디바이스 처리**: 영상 데이터가 네트워크를 통해 전송되지 않으므로 외부 노출 가능성이 구조적으로 차단됩니다.
- **HTTPS**: 모든 외부 통신(자동 업데이트·LLM 코칭·MediaPipe 모델)은 HTTPS 로 암호화됩니다.
- **자동 업데이트 서명 검증**: 모든 업데이트 패키지는 minisign 서명을 검증한 뒤 설치됩니다(위변조 방지).
- **localStorage 저장 한계**: 단일 사용자 데스크탑 환경을 가정하며, OS 사용자 계정 보안 수준에 의존합니다. API 키 등 민감 정보는 향후 OS 키체인 저장으로 보강 예정입니다.
