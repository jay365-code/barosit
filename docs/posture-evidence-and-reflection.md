# 자세 과학 근거 & BaroSit 반영 (Posture Evidence & Product Reflection)

> 작성일: 2026-06-22 · 출처: 딥리서치(22개 출처 → 92개 주장 → 25개 적대적 검증, 24 확정/1 폐기)
> 목적: "좋은 자세=단일 정렬"이라는 통념의 근거를 검증하고, 카피·임상 주장·기능에 반영.
> **이 문서는 앱 임상 카피(monitor.json 등)와 FAQ/랜딩 문구의 근거 출처(single source of truth)입니다.**

---

## 1. 핵심 결론 (TL;DR)

- **단 하나의 '완벽한 자세'는 과학적으로 검증된 적이 없다.** "표준 자세"는 19세기 가설에서 비롯된 신화에 가깝다.
- **특정 정렬(거북목·구부정 등)이 통증을 "유발"한다는 인과 합의는 없다.** 상관관계는 약하고 일관되지 않다.
- **진짜 문제는 정렬이 아니라 "한 자세를 오래 유지하는 것"** = 움직임 문제. "가장 좋은 자세는 다음 자세다."
- **오래 앉기의 위험은 활동량에 강하게 좌우된다.** 하루 60–75분 중강도 활동이 좌식 사망위험을 사실상 상쇄.
- **화면 리마인더는 실제로 좌식 시간을 줄인다**(하루 ~12분, 18 RCT 메타분석). → BaroSit 리마인더 기능의 직접 근거.
- ⚠️ **PRI(비대칭/Left AIC) 이론은 검증되지 않음** — 주류 근거 부족, 마케팅에 인용 금지.

### 한계 / 미해결
- 자세-통증 근거 다수가 단면연구(역인과 가능: 통증→자세). 일부 핵심 출처는 서술적/단일저자 리뷰.
- 변동성 증가 개입의 RCT 통증 결과는 불충분. 웨어러블 실시간 피드백 효과 직접 RCT 미확인.
- **PRI(Postural Restoration) 비대칭/Left AIC**: 1차 근거 미확보 → 별도 표적 리서치 필요. 현재 마케팅 인용 금지.

## 3. 통념 vs 근거 → 카피 원칙

| 통념(피해야) | 근거 기반 표현(사용) |
|---|---|
| "거북목이 통증을 유발한다 / 27kg 하중" | "머리가 앞으로 나오면 정적 부하가 커지기 쉬운 자세" (인과 단정 X, 수치는 모델 추정치로만) |
| "완벽한 자세를 유지하세요" | "단 하나의 완벽한 자세는 없다 — 자주 바꾸는 것이 핵심" |
| "잘못된 자세 = 위험" | "같은 자세를 오래 유지하는 것이 문제 — 잠깐 움직이세요" |
| "디스크 3배 / 통증 3.1배 / 젖산 완전 방지" 등 단정 수치 | 삭제. 검증된 출처(위 표)만 인용 |

## 4. 반영 내역 (P0–P2, 2026-06-22)

### P0 — 임상 주장 보정 (완료)
`src/i18n/locales/{ko,en,ja}/monitor.json`에서 과장·허위 인과 주장 제거:
- `stats.violations.ground`: "하버드…통증 3.1배" → 정적 피로 + 자주 움직이기
- `stats.stretch.ground`: "젖산 완전 방지/산소 25%" → 50분마다 자세 바꾸기(KOSHA)
- `stats.usage.ground`: "4.2배 급증" → 리마인더 효과(RCT 메타분석)
- `criteria.dDesc` / `grade.D.desc` / `legendD`: "27kg 돌파/만성통증 위험" → "정적 부하 누적 패턴"
- `report.basisBody`: Nachemson·NASA는 "참고 모델", 수치는 "추정치", "완벽 입증" 삭제, "완벽한 자세는 없다" 추가
- `report.clinicalWarn.*` / `report.biomechDetail.*` (각 7종): 질병 인과 동사 → "부하·긴장이 쏠리기 쉬운 패턴, 자주 바꾸기"
- `aiGuide.*`: "관절염 예방/디스크 절반/손목터널 유발" 과장 완화

### P1 — 강조점 재배치 (완료)
- 랜딩 features에 근거 강한 2기능 노출 (`Marketing.tsx` + `marketing.json` ko/en/ja):
  - `breaks`(오래 앉기 방지 리마인더, stretch 아이콘), `variability`(자세 변동 유도, bell 아이콘)
- `landing.hero.desc`: "완벽한 자세 강요 안 함 — 자주 움직이도록 돕는 게 핵심" 추가
- FAQ에 `goodposture` 항목 추가(ko/en/ja) + `index.html` FAQPage 구조화 데이터 동기화
- `index.html` SoftwareApplication description: "교정" → "오래 앉기 줄여 자주 바꾸도록" 보정
- 트래커 기본값 점검: BreakTracker/VariabilityTracker/CumulativeLoadTracker/AdaptiveSensitivity **모두 enabled:true** (변경 불필요)

### P2 — 포지셔닝 (후속)
- `docs/마케팅전략.html`에 "움직임 코치" 차별화 반영 (경쟁사 "완벽 자세 강박" 대비) — 미착수
- PRI·웨어러블 피드백 보강 리서치 후 별도 결정

## 5. 향후 인용 규칙
- 임상/건강 주장은 반드시 위 §2 표의 출처 범위 안에서만. 새 수치·배수 단정 금지.
- "유발/예방" 같은 인과 동사는 근거가 인과를 입증한 경우에만. 대부분은 "연관/경향/쉬움"으로.
- PRI, "27kg", "3.1배" 등 폐기된 표현 재도입 금지.
