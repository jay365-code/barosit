# BaroSit — 데스크톱 자세 모니터링 앱 기획 (현행)

> 본 문서는 **현재 구현된 앱**을 기준으로 한 기획 정리. 초기 기획안에서 실제 개발 과정 중 다수의 결정이 갱신·확장됨.

## Context

장시간 컴퓨터 사용자의 자세성 건강 문제(거북목·턱 괴임·어깨 기울임·등 구부정)를 데스크톱 웹캠으로 실시간 감지하고 알림 + 점수로 자세 교정을 유도하는 백그라운드 데스크톱 앱. 카메라 영상은 절대 외부로 나가지 않는 **온디바이스 처리**가 핵심 가치이며, **잘못된 알림(false positive) 최소화 + 자세 교정 동기부여**가 두 축. LLM 기반 맞춤형 코칭 메시지는 옵션 기능.

## 핵심 결정사항

| 항목 | 결정 |
|---|---|
| 플랫폼 | 데스크톱 앱 (Tauri 2) |
| 자세 분석 위치 | 100% 온디바이스 (영상 절대 외부 전송 금지) |
| LLM | 옵션 — Anthropic Claude Haiku 4.5 (자세 텍스트 메타데이터만 전송) |
| 감지 자세 | 7종 위반(거북목·턱괴임·어깨기울임·등구부정·모니터과근접·어깨비대칭·머리기울임) + 7종 스트레칭 보상(기지개·양손머리뒤·어깨스트레치·사이드·어깨으쓱·목좌우·앞숙이기) |
| 알림 방식 | OS 네이티브 알림 + 트레이 아이콘 상태 표시(이모지 색상) |
| UX 모드 | 메인 모드(큰 창) ↔ 위젯 모드(작은 카메라 아이콘 + 미니바) |
| 동기부여 장치 | 0–100 점수 시스템 + 스트레칭 보너스 + 빠른 회복 보너스 |
| 데이터 저장 | localStorage (이벤트 이력·점수·설정·베이스라인) — 단일 사용자 데스크톱 앱이라 충분 |
| 최우선 가치 | 알림 정확도 + 백그라운드 안정성 |

## 기술 스택

- **앱 프레임워크**: Tauri 2 (Rust 백엔드 + macOS WKWebView)
  - 다중 윈도우 (메인 + 위젯) 지원
  - `macOSPrivateApi: true` 로 위젯 투명 배경
- **프론트엔드**: TypeScript + React 18 + Vite
- **자세 감지**: `@mediapipe/tasks-vision` 4개 모델 동시 사용
  - **Pose Landmarker** (33점) — 어깨/팔/머리 기본 골격
  - **Face Landmarker** (478점 + 변환 행렬) — pitch/yaw/roll 추출
  - **Hand Landmarker** (21점 × 2) — 손가락 끝 위치 (턱괴임 정확도)
  - **Image Segmenter** (selfie_multiclass) — 프라이버시 모드 실루엣
- **카메라**: 웹뷰 `getUserMedia` (256×192, GPU 부담 최소화)
- **저장소**: localStorage (윈도우 간 자동 동기화)
- **LLM**: Anthropic Claude `claude-haiku-4-5` (옵션) + prompt caching
- **Tauri 플러그인**: `notification`, `store`, `autostart`, `single-instance`

## 자세 감지 로직

### 7가지 위반 자세

> 코드 기준([src/pose/types.ts](../src/pose/types.ts), 2026-07-08): 7종.

1. **거북목 (forward_head)** — 4신호 합산
   - 얼굴-어깨 너비 비율 변화 (가장 신뢰도 높음)
   - Pose z 좌표 변화
   - 코의 y 좌표 변화
   - Face Landmarker pitch 변화

2. **턱 괴임 (chin_resting)** — 3경로 중 하나
   - Pose 좌: 손목이 얼굴 근처 + 앞팔이 위로 향함
   - Pose 우: 동일 (반대편)
   - Hand: 손가락 끝이 코 근처

3. **어깨 기울임 (shoulder_tilt)**
   - `|ls.y - rs.y - baseline.shoulderTiltY|` > 0.04 × sensitivity

4. **등 구부정 (slouching)**
   - 어깨 너비 좁아짐 + 어깨 중심 y 내려감 합산

5. **모니터 거리 과근접 (monitor_too_close)**
   - -zDelta(얼굴이 카메라로 다가옴) + 귀 너비 비율 확대

6. **어깨 비대칭 (shoulder_asymmetry)**
   - 부호 있는 tilt + 어깨 중점 대비 코 x 오프셋 변화

7. **머리 좌우 기울임 (head_roll)**
   - 머리 roll 각도(좌우로 갸웃)

### 7가지 스트레칭 (보너스)

> 코드 기준([src/pose/stretchDetector.ts](../src/pose/stretchDetector.ts) `StretchKind`): 7종.

| 종류 | 검출 | 보너스 |
|---|---|---|
| 기지개 (overhead) | 양 팔꿈치가 어깨 위로 25% 이상 | +5 |
| 양손 머리 뒤 (behind_head) | 양 손목이 양 귀 근처 | +5 |
| 어깨 스트레치 (cross_body) | 한 손목이 반대편 어깨 너머 | +4 |
| 사이드 굽힘 (side) | 한 팔꿈치만 어깨 위 + 어깨 기울임 | +3 |
| 어깨 으쓱 (shoulder_shrug) | 양 어깨가 baseline 대비 명확히 상승 | +3 |
| 목 좌우 기울이기 (neck_side) | 머리를 좌/우로 기울임 | +3 |
| 상체 앞 숙이기 (forward_fold) | 상체를 앞으로 숙임 | +3 |

2초 유지 + 종류별 60초 쿨다운. (보너스 점수는 코드 기준으로 재확인 필요)

## False Positive 최소화 전략

1. **개인 캘리브레이션** — 5초 측정 + 5가지 적합성 체크(상체 보임/고개 들기/머리 좌우 수평/손 떼기/자세 유지) 65% 이상 적합 시 베이스라인 저장. 측면 카메라(노트북 옆 모니터)도 그 각도가 기준이 됨.
2. **EMA + 히스테리시스** — 매 프레임 신호에 0.15 EMA, 진입 임계 0.6 / 이탈 임계 0.3 + 최소 3초 유지. 미세 흔들림으로 인한 status ping-pong 제거.
3. **점수 그레이스** — 위반 시작 후 2초까지는 점수 감점 0 (자연 움직임 보호).
4. **신뢰도 필터** — MediaPipe `visibility < 0.7`인 키포인트는 해당 프레임에서 무시 (스트레칭은 0.35로 완화).
5. **이동평균 스무딩** — Pose 랜드마크 직전 7 프레임 이동평균.
6. **알림 쿨다운** — 같은 종류 알림은 5분 내 재발 안 함.
7. **부재 감지** — 8초 이상 사람 미감지 시 모니터링 자동 일시정지.
8. **사용자 피드백 신고** — 위젯 호버 패널의 "잘못된 알림" 버튼으로 해당 자세 sensitivity +0.1 (덜 민감해짐, 최대 2.0).

## 점수 시스템

0–100 점수, localStorage 영속화. 윈도우 간 storage 이벤트로 자동 동기화.

### 위반 시 패널티 (지속 시간 가속)
| 지속 | 초당 |
|---|---|
| 0–2초 | 0 (그레이스) |
| 2–10초 | −0.5 |
| 10–30초 | −1 |
| 30–60초 | −2 |
| 60초+ | −3 |

동시 위반 N개는 합산.

### 좋은 자세 회복 (연속 시간 가속)
| 좋음 지속 | 초당 |
|---|---|
| 0–5분 | +1 |
| 5–15분 | +2 |
| 15분+ | +3 |

**빠른 회복 보너스**: 위반 해제 후 10초 이내면 일회성 +2.
**스트레칭 보너스**: 위 7종 표.

## UX 모드 시스템

두 모드 mutually exclusive — 한 시점에 하나만 활성.

### 메인 모드 (기본)
- 메인 창 보임 — 모니터·캘리브·대시보드·설정 탭
- 미니바 (플로팅 윈도우) 보임 — 설정으로 끌 수 있음
- 검출 owner: 메인의 MonitorView
- 큰 실루엣 + 점수 chip + 위반 chip 표시

### 위젯 모드
- 메인 창 숨김
- 플로팅에 **미니바 + 카메라 아이콘** 함께 표시
- 검출 owner: 플로팅의 `useMonitoringEngine`
- 백그라운드 모니터링 — 메인 닫혀도 모니터링 유지

### 모드 전환

- **메인 X 클릭** → Rust `CloseRequested` 가로채기 → JS가 자동으로 위젯 모드 전환
- **Dock 아이콘 클릭** → `RunEvent::Reopen` → 메인 표시 + 메인 모드 자동 전환
- **메인의 "위젯 모드로 전환" 버튼** → 명시적 전환
- **카메라 아이콘 클릭** → 메인 모드로 복귀
- **트레이 아이콘 클릭** → 메인 표시
- 250ms 핸드오버 지연 + 카메라 실패 시 800ms 후 재시도 (race condition 방지)

## 시각화 (프라이버시 모드)

기본 ON — 카메라 영상 대신 추상화된 실루엣:
- Image Segmenter 마스크로 사람 영역만 색칠 (상태색)
- Face mesh 478점이 얼굴 위 점으로
- 어깨 가로 라인 표시
- 멀티레이어 블러로 부드러운 글로우
- 포즈 기반 클리핑으로 의자 등 주변 사물 제거

## 종료 / 라이프사이클

- **닫기 X** → 트레이 숨김 + 위젯 모드 자동 전환 (앱은 살아있음)
- **트레이 메뉴 "종료"** → 완전 종료
- **설정 화면 "앱 종료" 버튼** → 두 번 클릭 패턴으로 종료 확인
- **macOS Cmd+Q** → 시스템 단축키
- **단일 인스턴스 락** → 두 번째 실행 시 첫 인스턴스 메인 창 띄움

## LLM 코칭 (옵션)

설정에서 API 키 입력 시 활성. 알람 발사 시:
- 자세 종류, 지속 시간, 시간대, 오늘 누적 횟수 텍스트만 전송
- Claude Haiku 4.5 + prompt caching
- 응답을 추가 알림으로 전송
- 실패/오프라인 시 디폴트 메시지

## 검증 시나리오

- [ ] 캘리브레이션 후 거북목 자세 → 5초+ 후 알람

## 플랫폼 확장 전략

### Windows 버전 (강력 추천)

같은 Tauri 코드베이스로 동작. 처리 필요한 OS 차이:

| 항목 | 차이 |
|---|---|
| `macOSPrivateApi` | Windows에서 자동 무시 — 위젯 투명은 Windows 별도 처리 |
| 트레이 메뉴 | 시스템 트레이 영역, 메뉴 UX 다름 |
| `RunEvent::Reopen` | macOS 전용 — Windows에선 트레이 클릭으로 대체 |
| 알림 | Action Center — `tauri-plugin-notification`이 자동 처리 |
| 단축키 | Cmd+Q → Alt+F4 등 시스템 단축키 매핑 |
| 외장 웹캠 | Windows 사용자 비율 높음 — 호환성 검증 필요 |

**가치**: 시장 크기 2~3배 확장, 핵심 가치(백그라운드 모니터링) 그대로 유지.

### Web 버전 (제한적 — 데모 용도 권장)

#### 근본 한계

웹 환경에서 못 하는 것들:
- ❌ 백그라운드 모니터링 (탭 닫히면 정지)
- ❌ 시스템 트레이 아이콘
- ❌ 항상 위 떠있는 위젯 (브라우저 안에 갇힘)
- ❌ OS 글로벌 단축키
- ❌ 다른 앱 위에 노출 불가

#### 적합한 용도
- **데모/트라이얼 진입 장벽 제거** — 다운로드 없이 5초 체험
- 마케팅 funnel (랜딩 페이지에서 바로 측정 → "백그라운드 모니터링은 데스크톱 앱에서" CTA)
- 회사 PC 등 설치 불가 환경의 가벼운 일시 사용
- 소셜 공유 가능한 URL

#### 작업 범위 (lite 데모 기준)
- IPC 추상화 레이어 (Tauri vs Web 구현 분기)
- 트레이/네이티브 알림 → 브라우저 Notification API
- 다중 윈도우 → 단일 페이지 + 모달
- 모드 시스템 → 메인만 유지 (위젯 모드 제거)
- 검출 파이프라인 자체는 그대로 (MediaPipe는 브라우저 동작)

**포지셔닝**: 풀 서비스 아닌 **마케팅 도구**. 핵심 가치는 데스크톱 앱에 보존.
